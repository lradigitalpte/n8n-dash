Zumbaton WhatsApp Automation — Technical Documentation

Single-file overview for stakeholders and engineers. Open this file in Microsoft Word (File → Open) to edit or export to PDF. Last updated: April 2026.

1. Executive summary

This system connects Twilio WhatsApp to an AI assistant, persists conversations in Convex, and provides an operator dashboard (Next.js) for leads, knowledge base content, and optional human takeover. Orchestration runs in n8n (workflow automation). The dashboard is typically deployed on Vercel; Convex runs on Convex Cloud; n8n runs wherever Twilio can reach it (self-hosted, n8n Cloud, etc.).

3. n8n — what it does and how it works

n8n is a visual workflow engine. Each inbound WhatsApp message fires a workflow that:

  • Normalizes input — Maps Twilio fields (from, to, body, media) into a consistent shape and holds configuration: Convex base URL, shared HTTP secret, org id, owner phone for alerts.
  • Conversation state — Calls Convex /getStatus to see if the thread is in human takeover (set from the dashboard). If active, the path can save the user message and optionally send a short “team is handling this” reply without running the full LLM.
  • Media branch — Routes images/voice/other attachments to fallbacks or simple replies.
  • Chat path (default) — Loads recent history from Convex (/getHistory), searches the knowledge base (/searchKB), builds a single prompt (system + KB snippets + history + latest message), calls the LLM (e.g. OpenRouter), then parses the reply.
  • Lead capture — If the model returns a structured [LEAD]{...}[/LEAD] block, the workflow can POST to /ingestLead, send email/SMTP, notify the owner, and confirm to the user.
  • “Needs handover” detection — Keyword/heuristic logic flags when the user asks for a human or seems stuck. The workflow can notify the owner on WhatsApp with context but (by design) does not automatically flip Convex to human takeover — so the bot can keep helping until an operator takes over from the dashboard.
  • Persist messages — Saves user and bot lines via /saveMessages so the dashboard and future turns stay in sync.

3.1 n8n use cases (business)

  • 24/7 first-line answers using studio-approved KB (pricing, schedule, policies).
  • Capturing leads (name, email, phone) into Convex for follow-up.
  • Alerting the owner when someone wants a human or submits a lead.
  • Graceful handling of media and edge cases without crashing the flow.

3.2 n8n — technical notes

  • All bot → Convex HTTP calls should use the *.convex.site base URL (HTTP actions), not the dashboard websocket URL (*.convex.cloud).
  • If WHATSAPP_BOT_HTTP_SECRET is set in Convex, every request must send the same secret (query, header, or JSON body) or routes return 401 — including /searchKB. A failed KB call leaves the model without facts (e.g. “no pricing”).
  • The workflow version discussed here returns KB as { "chunks": [...] } in one JSON body so n8n does not split arrays into many items incorrectly.

4. Human handover — behaviour

Convex conversation status human_takeoverDashboard (operator)n8n reads /getStatus; user can get a holding message; bot automation is reduced according to workflow design.
User says “speak to someone”Customern8n can notify the owner with chat context; does not automatically set Convex takeover — customer can keep chatting with the bot until staff take over from the UI.
Outbound replies from staffDashboard queue + n8n bridgeHuman-written messages can be pulled/marked sent via Convex HTTP so Twilio sends from the same number.

Design intent: Avoid promising “someone will message you immediately” when no one is available. The bot remains available for factual questions until a human explicitly takes the thread in the dashboard.

5. Dashboard — features (Next.js)

  • Authentication — Better Auth with Convex; production SITE_URL must match the deployed app URL (e.g. Vercel). Optional extra origins for local development.
  • Leads — List/filter leads ingested from the bot (org-scoped).
  • Inbox / conversations — Live thread view backed by Convex; handover toggle; queue for human replies to WhatsApp.
  • Knowledge base — Manage chunks used by n8n’s /searchKB for RAG-style prompts.
  • Settings / org — Align orgId with n8n defaultLeadOrgId and Convex data.

8. Glossary

  • RAG — Retrieval-augmented generation: KB chunks are injected into the prompt so answers follow studio facts.
  • Org id — Logical tenant (e.g. org_zumbaton) for leads, conversations, and KB rows.
  • Human takeover — A Convex flag on the conversation, controlled from the dashboard, not automatically by “I want a human” alone (in the described design).
